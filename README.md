## サマリ

業務システム、SaaS、CRM領域を横断して経験。  
レガシー環境の調査・改善、SaaSプロダクトの開発・運用、スクラムマスターとしてのチーム改善、Salesforce/CRM Analytics の実装まで担当。  
散逸した情報の再構造化や、チーム内のプロセス改善、ドキュメント整備など「仕組みとして残る改善」が得意。

## 強み

- 技術・組織・業務の文脈を抽出して構造化する力  
- レガシー環境の調査（考古学的アプローチ）と現代的開発フローへの再構成  
- スクラム運用を基軸としたチーム改善、障害除去、ファシリテーション  
- 属人化したナレッジや情報を再整理し、再現性のあるドキュメントに落とす能力  
- 未知領域でも仮説検証しながら自走し、立ち上げ段階のカオスを整理すること

## 職務経歴

### 2009/04 – 2019/06： SIer（業務システム保守・開発）

#### 物流機器メーカー向け業務システム（OracleEBS / intra-mart）
- 保守・運用および小規模改修（問い合わせ対応、日〜週単位の開発）
- intra-mart による画面／ワークフロー開発
- 既存システムの調査、仕様確認（ドキュメントとコードの乖離調整を含む）

#### システムリプレース（OracleEBS → intra-mart）
- 最新バージョンへの再構築プロジェクトに参画  
- 詳細設計〜実装、単体試験を担当  

#### 社内業務システムのリプレース
- 古い intra-mart から新バージョンへの移行  
- 実装可否調査、試行コード作成、要件定義〜結合試験まで一通り担当  

NOTE: この期間は大規模業務システムの構造理解、レガシーコードの調査、現場運用に近い改善対応が中心。

### 2019/07 – 2021/04：建築向けSaaSベンチャー

#### SaaSプロダクトの開発・改善
- バグ対応、追加改修、営業向け機能、グロース機能の実装
- 開発が止まった自社サービスの復旧（環境構築、散逸した情報の再整理）

#### 改善・整備した領域
- **開発環境の再構築**：動かなくなっていた環境を再調査し構築し直す  
- **ナレッジ統合**：ドキュメント・仕様・コードの乖離を整理  
- **UI/UX改善**：集計管理機能、メール一斉送信機能などを追加  
- issue 起票〜リリースまで一人で一貫対応し、意思決定〜実装のサイクルを短縮  

NOTE: 技術的負債の整理や、立ち上げ期のカオスの整頓が中心。

### 2021/06 – 2024/01：人事労務向けSaaSベンチャー

#### エンジニア兼スクラムマスター
- スクラムイベントの運営、障害除去、チーム状況の可視化  
- メンバーの提案を採用しやすいプロセスを設計し、実験文化を育てる  
- 技術的課題・心理的課題の両面からチームの生産性を改善  

#### プロセス・ナレッジ改善
- **環境構築手順を抜本見直し**：更新条件の明文化、構築手順の再構造化  
- **情報散逸防止の仕組み化**：ドキュメント管理ルールを整備し、チームで維持可能に  
- **属人作業の分解**：再現性のあるフロー化による負担軽減  

NOTE: この時期に「チームの総出力を最大化する役割」が明確に発揮された。
